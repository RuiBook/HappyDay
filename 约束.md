前端项目约束文档
1. 项目结构规范
1.1 组件文件组织
每个组件必须有独立的文件夹
组件文件夹内必须包含组件文件和对应的SCSS样式文件
组件和样式文件必须在同一目录下

```
src/
  components/
    Card/
      Card.tsx
      Card.scss
    Header/
      Header.tsx
      Header.scss
  pages/
    Home/
      Home.tsx
      Home.scss
    Gallery/
      Gallery.tsx
      Gallery.scss
```
1.2 命名规范
组件文件名采用 PascalCase：ComponentName.tsx
样式文件名与组件名保持一致：ComponentName.scss
文件夹名称与组件名称保持一致
2. 样式文件规范
2.1 SCSS文件要求
每个组件/页面必须有独立的SCSS文件
禁止使用内联样式
使用BEM命名规范或CSS Modules

2.2 SCSS层级规范
2.2.1 最大嵌套层级限制
最多嵌套3层，避免过深的选择器嵌套
使用BEM命名减少嵌套需求
scss


```
// ❌ 错误示例 - 嵌套过深
.card {
  .header {
    .title {
      .icon {
        .svg {
          // 嵌套太深，难以维护
        }
      }
    }
  }
}

// ✅ 正确示例 - 扁平化结构
.card {
  &__header {
    // 第一层
  }
  
  &__title {
    // 第一层
  }
  
  &__icon {
    // 第一层
    svg {
      // 第二层，仅在必要时
    }
  }
}
```

3. 配置化管理
3.1 配置文件结构
创建统一的配置文件 src/config/display.config.ts：


```
export interface CardConfig {
  id: string;
  image: string;
  backgroundImage?: string;
  title: string;
  description?: string;
}

export interface DisplayConfig {
  cards: {
    count: number;
    items: CardConfig[];
  };
  assets: {
    defaultImage: string;
    defaultBackground: string;
  };
}

export const displayConfig: DisplayConfig = {
  cards: {
    count: 6,
    items: [
      {
        id: 'card-1',
        image: '/assets/images/card-1.jpg',
        backgroundImage: '/assets/backgrounds/bg-1.jpg',
        title: 'Card Title 1',
        description: 'Card description'
      },
      // 更多卡片配置...
    ]
  },
  assets: {
    defaultImage: '/assets/images/default.jpg',
    defaultBackground: '/assets/backgrounds/default.jpg'
  }
};
```
3.2 配置使用示例

```
// Gallery.tsx
import { displayConfig } from '@/config/display.config';
import Card from '@/components/Card/Card';

const Gallery = () => {
  const { cards } = displayConfig;
  
  return (
    <div className="gallery">
      {cards.items.slice(0, cards.count).map((card) => (
        <Card
          key={card.id}
          image={card.image}
          backgroundImage={card.backgroundImage}
          title={card.title}
          description={card.description}
        />
      ))}
    </div>
  );
};
```

6. 开发规范检查清单
 组件有对应的SCSS文件
 组件和样式文件在同一文件夹
 SCSS嵌套不超过3层
 SCSS文件结构清晰有序
 使用BEM命名规范
 使用配置文件管理卡片数量
 使用配置文件管理图片资源
 没有硬编码的图片路径
 遵循命名规范
7. 示例组件结构

```

// components/Card/Card.tsx
import './Card.scss';

interface CardProps {
  image: string;
  backgroundImage?: string;
  title: string;
  description?: string;
}

const Card: React.FC<CardProps> = ({ image, backgroundImage, title, description }) => {
  return (
    <div 
      className="card"
      style={{ backgroundImage: backgroundImage ? `url(${backgroundImage})` : undefined }}
    >
      <img className="card__image" src={image} alt={title} />
      <div className="card__content">
        <h3 className="card__title">{title}</h3>
        {description && <p className="card__description">{description}</p>}
      </div>
    </div>
  );
};

export default Card;
```

8. 配置更新流程
修改 display.config.ts 文件
更新卡片数量：修改 cards.count 值
替换图片：修改对应的 image 或 backgroundImage 路径
添加新卡片：在 cards.items 数组中添加新的配置项
页面自动根据配置渲染
